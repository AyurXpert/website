1. Overview

This guide covers subscription.html — the single page a platform admin uses to manage every tenant's plan, billing, NCISM capacity requests, and the platform-wide pricing/GST configuration that every tenant sees.

1.1 Who Is Platform Admin

The platform_admin role is separate from every tenant's own staff roles (super_admin, dept_admin, etc.) — it is the AyurXpert company's own account, not owned by any hospital tenant. On login it lands directly on subscription.html and is gated by the is_platform_admin boolean on the profiles table (checked live on every page load, never cached).

Only one platform_admin account should normally exist As of this writing there is exactly one: admin@ayurxpert.com. If a second one is ever created, both would see and manage the same tenant-wide data — there's no per-admin scoping.

2. All Tenants Tab

The default tab on subscription.html. Lists every tenant on the platform with their plan, billing status, and usage.

2.1 Summary Stats

Five cards at the top of the page: Total Tenants, On Trial, Active Paid, Expiring ≤30d, Expired. These recompute from the same tenant list shown below — they are not separately cached.

2.2 Search & Filter

Filter the tenant table by organisation name (text search), subscription status, or plan type. Filters combine with AND logic — e.g. status=Expiring + plan=Growth shows only Growth-plan tenants that are expiring.

2.3 Editing a Tenant's Subscription

Click ✏ Edit on any tenant row to open the Edit Subscription modal. This is the only place a platform admin directly changes a tenant's billing state.

2.4 Expiring / Expired Tenants

The tenant table automatically sorts Expired tenants first, then Expiring tenants (soonest first), then everyone else alphabetically — so tenants needing attention are always visible at the top without needing to apply a filter. Expired/expiring rows also get a light red/amber background tint.

3. NCISM Requests Tab

Teaching Hospital / College tenants submit a capacity request from their own admin.html Subscription tab (new UG intake tier and/or PG department seats). Every pending request appears here for review.

  1. 1
    Review the request — organisation name, current vs. requested UG intake, requested PG department additions, and the computed fee (including its GST breakdown, frozen at the rate that applied when the tenant submitted).
  2. 2
    ✓ Approve — confirms, then immediately runs platform_approve_ncism_request(): seeds OPDs/departments/beds on a tenant's very first activation, or adds only the positive bed/seat delta on a later upgrade (never removes existing beds).
  3. 3
    ✗ Reject — prompts for an optional reason, marks the request rejected. No data changes on the tenant.
Approval is immediate and irreversible via the UI There is no "undo approve" button. If a request was approved by mistake, the tenant's OPDs/departments/beds must be corrected manually (or via a fresh SQL fix) — there's no automated rollback path.

4. Pricing Tab

Every ₹ figure a tenant ever sees anywhere in the app — NCISM capacity fees, and (once set) the SaaS subscription plan fees — is configured from this one tab. Nothing here is hardcoded in the app's code.

4.1 GST Rate

A single platform-wide GST percentage, applied on top of every fee shown anywhere (NCISM tiers, PG seat fee, SaaS plan pricing). It is intentionally a single editable number rather than hardcoded, since the government rate changes from time to time. Changing it here immediately changes the "+GST Total" figures shown across this page, the tenant's own NCISM Capacity Plan card, and their Current Plan card — nothing needs to be redeployed.

GST is calculated, not stored, on most fee displays Only NCISM capacity requests freeze the GST rate/amount that applied at submission time (for audit correctness — a later rate change shouldn't silently rewrite a pending or historical request's fee). Every other fee display (tiers, PG seat fee, plan pricing) recomputes live from the current rate.

4.2 UG Intake Tiers

The fixed set of UG intake sizes (currently 60 / 100 / 150 / 200 students/year) a Teaching Hospital/College can choose between, each with its own one-time capacity-activation fee. Edit a tier's fee, or Deactivate a tier to hide it from tenants without deleting its history. "+ Add Tier" creates a new intake size if the institution needs one not already listed.

4.3 PG Seat Fee

A single per-seat fee applied to every PG department seat a tenant requests, regardless of department. There is only one PG seat fee for the whole platform (not one per department).

4.4 SaaS Plan Pricing

Independent monthly and annual fees for the Starter, Growth, and Enterprise plans. Free Trial has no pricing row — it is always ₹0 and has no billing cycle.

Annual is not auto-discounted from monthly Enter the actual annual figure you want to charge — the system does not compute it as "monthly × 12 minus a discount." This was a deliberate choice for flexibility.

Each plan row has its own Save button — editing Starter's monthly fee does not affect Growth or Enterprise, and editing monthly does not affect annual.

5. Common Scenarios

A. A tenant's trial is about to expire and they've decided to subscribe

  1. 1
    Go to All Tenants, find the tenant, click ✏ Edit.
  2. 2
    Set Plan to Starter/Growth/Enterprise, set Billing Cycle to Monthly or Annual, set Status to "Active (Paid)".
  3. 3
    Set Subscription Expiry to one month or one year out (matching the billing cycle you chose).
  4. 4
    Save. The tenant's own dashboard immediately shows their new plan, billing cycle, and fee (with GST) on next load.

B. A Teaching Hospital wants to increase their UG intake mid-year

No action needed from the platform admin until the tenant submits a request themselves from their own Subscription tab. Once submitted, review and approve it from the NCISM Requests tab (Section 3).

C. The government changes the GST rate

Go to Pricing → GST Rate, update the number, click Save. Every fee display across the platform (this page and every tenant's dashboard) reflects the new rate immediately — no other changes needed.

6. Troubleshooting

I saved a fee and got a success message, but the old value is still showing As of Session 89, every pricing save (GST rate, tiers, tier activation, PG seat fee, plan pricing) explicitly checks that a row was actually returned after saving. If you now see a real error message like "Save did not apply — no matching row", that's a genuine permissions/data problem — check that your account still has is_platform_admin = true. If you still see a false "success" with no change, try a hard refresh (Ctrl+Shift+R) first — a stale cached copy of the page's JavaScript is the most common cause.
A change I just made isn't visible on the live site yet The live ayurxpert.com site is a separate deploy from the app's source — every change needs to be synced and pushed before it's visible, and Cloudflare's CDN cache can take a minute or two to catch up even after that. Hard-refresh (Ctrl+Shift+R) before assuming something is broken.
