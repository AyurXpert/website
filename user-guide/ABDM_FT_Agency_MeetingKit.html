🏥 Application Overview

AyurXpert is a multi-tenant HMS (Hospital Management System) SaaS platform for Ayurveda healthcare facilities across India.
  • Each hospital / clinic / Panchakarma centre is a separate tenant with their own data, staff, and branding
  • Staff log in to a shared platform but see only their organisation's data (RLS-enforced)
  • Acts as both HIP (provides patient records to ABDM) and HIU (fetches records from ABDM for doctors)
  • Web-based Progressive Web App (PWA) — no native mobile app
  • Live URL: https://ayurxpert.com
  • Login: https://ayurxpert.com/login.html

Application Count for FT / WASA

1 web application — AyurXpert HMS at ayurxpert.com.
The Edge Functions (abdm-auth, abdm-webhook) are backend components of the same application, not separate apps.
No mobile app is in ABDM scope.

📋 M1 — ABHA (Patient Identity)

ABHA Creation & Enrollment

  • Aadhaar OTP-based ABHA creation via reception.html
  • ABHA number and ABHA address stored against patient record

4 Login / Verification Flows

  • Aadhaar OTP: POST /abha/api/v3/enrollment/auth/byAadhaar
  • ABHA Number: POST /profile/login/request/otp (loginHint: abha-number)
  • ABHA Address: POST /phr/web/login/abha/request/otp → verify → GET abha-profile
  • Mobile OTP: POST /profile/login/request/otp (loginHint: mobile) → accounts[] returned

Scan & Share

Patient scans QR from reception screen in ABHA app → patient data auto-fills in reception.html → token raised → visit created. Confirmed working in sandbox.

Profile / QR Code / ABHA Card

  • GET /profile/account — full profile + KYC ✅
  • GET /profile/account/qrCode — 202 ✅
  • GET /profile/account/abha-card — 202 ✅

📋 M3 — HIU (Health Record Fetching)

Consent Flow

  • Doctor initiates consent request from doctor.html Tab 9 (ABDM Records)
  • Patient receives notification on ABHA app → approves
  • Consent callbacks handled: grant / deny / revoke / expire

Data Fetch & Display

  • Fetch consent artefact → data request → HIP pushes encrypted FHIR
  • Decrypt using P-256 ECDH + AES-256-GCM
  • Display: longitudinal timeline grouped by date (diagnoses, medications, vitals, labs, allergies)

Revoke / Expire

Consent revoked → records deleted from DB + compliance notice shown in UI. HIU_FLOW_202 ✅
