1. Overview

This guide covers the complete IPD Bed Setup module — every feature in bed-admin.html — from seeding NCISM-mandatory departments through physical bed configuration to the printable Bed Map PDF.

It is written for two audiences side-by-side: hospital administrators who need to understand the workflow, and developers/technical staff who need the database schema, constraints, and function references.

1.1 What It Does

The IPD Setup page is the configuration layer for all inpatient operations. Before any patient can be admitted via ipd.html, the following must be in place:

  • At least one department must exist with an active NCISM code
  • At least one bed must be assigned to that department with status vacant
  • For NCISM-accredited colleges: the Table-8 minimum bed count must be met per department

The page provides three paths to achieve this: manual department-by-department setup, bulk bed creation, and the ⚡ Quick Setup tab which reads your physical inventory and distributes beds algorithmically.

Other roles cannot access this page. requireAuth(['super_admin', 'dept_admin']) is enforced server-side. A nurse or receptionist visiting bed-admin.html directly will be redirected to their home page.

1.3 How to Access

Three routes lead to this page:

  • Navbar → Admin → IPD Setup (visible only to super_admin / dept_admin)
  • admin.html sidebar → Departments section → "Manage Beds →" link
  • Direct URL: /bed-admin.html

The icon-rail sidebar (52 px strip on the left of all admin pages) also has an IPD icon that navigates here.

3. Departments Tab

The Departments tab is the primary management view. It shows one card per department, sorted by NCISM required beds in descending order (highest compliance obligation first).

3.1 Department Card Anatomy

Each card contains:

  • Department name — full name as entered
  • NCISM code — e.g. PANCHAKARMA, shown below the name
  • Badges — NCISM (mandatory dept), PG (postgraduate department), Ratio⚠ (compliance violation), Active/Inactive
  • NCISM Bed Row — "NCISM Required: 15 / Configured: 15 / ✓ Beds met" or "⚠ Need 6 more"
  • Dept Stats — Beds count, Occupied count, Occupancy %, PG beds (for PG depts)
  • Bed chips — one chip per bed showing bed number + floor + type abbreviation; click any chip to open the edit drawer
  • Action buttons — + Bed / 🗑 All Beds (if beds exist) / Edit (✎) / Delete (🗑)
Sort order: Cards are sorted by NCISM required beds descending: Panchakarma (25%) → Kayachikitsa (20%) → Shalya Tantra (20%) → Stri Roga (10%) → Kaumarabhritya (10%) → Shalakya Netra (5%) → Shalakya KNM (5%) → Agada Tantra (5%) → non-NCISM departments (alphabetical). This order ensures the highest-compliance departments are always visible first.

4.2 Bulk Add Beds

Click Bulk Add Beds in the page header. This opens a full-width drawer where you can select a department and then pick multiple bed numbers at once using a chip grid.

  • Select All — selects all available (unallocated) chips in the NCISM reserved range
  • Clear — deselects all
  • Extra beds section — beds beyond the NCISM quota are shown separately below the reserved range chips

All selected beds inherit the same floor, ward name, bed type, status, and PG flag from the form fields. If you need different floors for different beds, add them in separate bulk batches or use Quick Setup.

4.3 Edit a Bed

Click any bed chip on a dept card, or click a bed card in the Bed Matrix. The same drawer opens in edit mode with all fields pre-filled.

Bed Number is read-only for NCISM reserved-range beds. The number is locked (greyed, 🔒 note) to preserve NCISM compliance integrity. You can change floor, ward name, type, status, and PG flag freely. Extra beds (beyond the quota) keep their number editable (✏ note shown).

4.4 Delete Beds

Two deletion paths:

  • Single bed — open the edit drawer → Delete button at the bottom. Blocked if status is occupied.
  • All beds in a dept — 🗑 All Beds button on the dept card (red outline; appears only when the dept has beds). Deletes all non-occupied beds in one Supabase query (.neq('status','occupied')). Occupied beds are skipped with a warning. The department record itself is preserved.
Occupied beds cannot be deleted. A bed with status occupied has a live IPD admission linked to it. Change the bed status to vacant first (via the edit drawer) — or discharge the patient via ipd.html which automatically sets the bed back to vacant.

6.2 Table 2 — NCISM Department Allocation

Click ↓ Auto-Distribute to Departments to populate Table 2. The system reads your Table 1 inventory and distributes beds proportionally to each NCISM department.

Table 2 shows one row per department with:

  • Dept — department name (sticky column; stays visible when scrolling right)
  • Required — NCISM Table-8 minimum (e.g. "15" for Panchakarma at 60-intake)
  • One column per active bed type — how many of each type are assigned to this dept
  • Total — sum of all type columns; shows ✓ in green or ⚠ in red if mismatched
  • Rooms / Wards — physical room count
  • Building Sources — which blocks and floors contribute to this dept's beds (e.g. "Main Building F1 GF (23) · Fl.1 (7)")
  • Floor — dropdown to select which floor to store beds on; pre-populated from Table 1
  • Create — button to create beds for this dept only
Cells are editable. The auto-distributed numbers are a starting point — you can edit any cell in Table 2 before creating. The row total validates live and turns red if the total doesn't match what was distributed. A row marked ✓ green is safe to create.

6.3 Auto-Distribute Logic

The distribution algorithm is block-first, proportional:

  1. Compute each building block's share of totalNeeded proportional to its pool size (beds entered in Table 1)
  2. Assign bed counts to each block using the largest-remainder method so the total is exact
  3. Within each block, distribute assigned beds across types proportionally to that block's type mix
  4. A rotating _blkRemOffset counter alternates which block wins remainder ties — prevents the first block from always getting extra beds

Result: if you enter 30 beds in Block 1 and 30 in Block 2, each department gets exactly 50% from each block, not 38/22 due to remainder accumulation.

6.4 Floor Assignment

Each bed type in each building block has its own floor number in Table 1 (the "Floor" sub-column). When beds are created:

  • Every bed of type X in block B gets floor_number = the value in the Floor cell for that type+block combination
  • If the floor cell is blank, defaults to 0 (Ground Floor)
  • Floor values are persisted to localStorage per tenant so they survive page refresh
Floor inputs are live. Changing a floor value in Table 1 immediately updates localStorage AND recalculates the Rooms/Wards column — you do not need to click a save button for floor values.

6.5 Create All

Click Create All to insert all rows whose totals are ✓ (green) in one batch. The system:

  1. Resets _blkRemOffset to 0 for reproducible distribution
  2. Fetches existing beds for each dept to calculate the next available bed number
  3. Assigns sequential numbers within the NCISM reserved range for each dept
  4. Inserts all beds to Supabase in parallel per dept-row
  5. Calls loadAll() to refresh all tabs after creation
Quick Setup does not delete existing beds. If you already have 15 PK beds and run Quick Setup again, it will add MORE beds starting from the next available number. Always check existing bed counts before running Create All a second time. Use 🗑 All Beds on the dept card to clear first if you need to start over.

8.3 PG Bed Requirements

NCISM requires separate PG beds for departments with sanctioned PG seats. The system tracks:

  • is_pg_dept = true on the department
  • pg_seats_sanctioned — number of PG seats; PG beds required = seats × 2 (industry standard)
  • is_pg_allocated = true on individual beds — shown as a blue dot in Bed Matrix
  • Dept card shows: "PG beds: 4/6" (allocated / required)

PG beds should be in the same department as the UG beds but are physically segregated (separate ward). Use the Ward Name field to label them distinctly (e.g. "PG Ward – KC").

9.3 RLS Policies

Both tables have Row Level Security enabled. The pattern is identical for each:

-- departments table (same pattern for beds)
CREATE POLICY "tenant_isolation" ON departments
  USING (
    tenant_id = (
      SELECT tenant_id FROM profiles WHERE id = auth.uid()
    )
  );

-- Separate INSERT/UPDATE/DELETE policies follow the same USING clause
-- SELECT is also filtered — a super_admin from Hospital A cannot see Hospital B's data

No anon access is granted to either table. Only authenticated users with a profiles row (therefore a tenant_id) can read or write bed/dept data.

9.4 Database Constraints

beds_bed_type_check

ALTER TABLE beds ADD CONSTRAINT beds_bed_type_check
  CHECK (bed_type IN (
    'male_general','female_general','general',
    'twin_sharing','semi_private','private','deluxe',
    'dormitory','icu','day_care','pk_treatment','observation'
  ));
This constraint was updated in Session 71 to add male_general and female_general types. If you are running on an older schema, run the ALTER TABLE above before using Quick Setup (it inserts these types).

beds_status_check (implicit)

-- Status is validated in JS before insert; a DB constraint can be added as:
ALTER TABLE beds ADD CONSTRAINT beds_status_check
  CHECK (status IN ('vacant','occupied','maintenance','reserved'));

NCISM config columns on tenants (run once)

ALTER TABLE tenants ADD COLUMN IF NOT EXISTS ug_intake int DEFAULT 60;

ALTER TABLE tenants ADD COLUMN IF NOT EXISTS opd_daily_target int DEFAULT 120;

ug_intake drives all NCISM Table-8 calculations. Set it per tenant: 60 / 100 / 150 / 200 for colleges; leave as 0 for pure hospitals (disables NCISM compliance rows).

10. Common Scenarios

A. First-Time Hospital Setup (60 UG intake college)

  1. 1
    Set UG intake — In Supabase Dashboard → Table Editor → tenants, set ug_intake = 60 for the hospital's row. Or run: UPDATE tenants SET ug_intake = 60 WHERE id = '…';
  2. 2
    Seed NCISM Departments — Click Seed NCISM Depts. 10 mandatory departments are created in seconds. Verify all 10 appear in the Departments tab.
  3. 3
    Open ⚡ Quick Setup tab — Count your physical beds by type and enter them in Table 1. Add building blocks matching your layout (e.g. "OPD Block GF" with 30 beds on Floor 0, "IPD Block Fl.1" with 30 beds on Floor 1).
  4. 4
    Verify Table 1 total ≥ 60 — The grand total cell shows ✓ Meets NCISM min (60 needed). If below 60, add more bed entries before proceeding.
  5. 5
    Auto-Distribute → review Table 2 — Click ↓ Auto-Distribute to Departments. Check that each dept's Total column matches the Required column (all rows green). Adjust cells if needed.
  6. 6
    Create All — Click Create All. All 60 beds are inserted. Return to Departments tab to confirm each dept card shows "✓ Beds met".
  7. 7
    Generate Bed Map PDF — Click Bed Map PDF. Print or save as PDF for your NCISM inspection file.

B. Adding PG Beds to Kayachikitsa

Hospital has upgraded — NCISM has sanctioned 6 PG seats for Kayachikitsa. PG beds (= 2 × seats = 12 beds) must be added and marked.

  1. 1
    Edit the dept — On the KC dept card, click ✎ Edit. Toggle "PG Department" ON; enter pg_seats_sanctioned = 6. Save.
  2. 2
    Add PG beds — Click + Bed on the KC card. Set Ward Name = "PG Ward – KC", Floor = 2 (if on 2nd floor), Bed Type = Private Room, Status = Vacant, PG Allocated = ✓. Save. Repeat for all 12 PG beds (or use Bulk Add with consistent settings).
  3. 3
    Verify — KC dept card now shows "PG beds: 12/12". Bed Matrix shows blue dots on all 12 PG beds.

C. Scheduling a Bed for Maintenance

Bed PK-07's mattress needs replacement. It should be unavailable for 2 days.

  1. 1
    Confirm vacant — Check PK-07 in the Bed Matrix. If it shows OCCUPIED, a patient is using it — do not change status.
  2. 2
    Open edit drawer — Click the PK-07 card. Change Status to maintenance. Save.
  3. 3
    Confirm visual — PK-07 card in Bed Matrix now shows a grey MAINTENANCE pill. The bed disappears from "Vacant" filter results so admissions staff won't accidentally allocate it.
  4. 4
    Restore — After maintenance is complete, open the drawer again, set Status back to vacant. Card returns to green.

D. Monthly NCISM Compliance Audit

Prepare for a surprise NCISM inspection by verifying bed compliance across all departments.

  1. 1
    Open Departments tab — Scan the dept cards. Any card with a red "⚠ Ratio" badge or "⚠ Need N more" text is non-compliant.
  2. 2
    Cross-check with ncism-compliance.html — The NCISM Compliance Monitor page provides the same data in a 10-column table with occupancy percentages and CSV export. Use this for formal documentation.
  3. 3
    Generate Bed Map PDF — Click Bed Map PDF to produce a timestamped printout for the inspection file.
  4. 4
    Fix gaps — For any dept with a gap, use the Bulk Add Beds flow to fill the remaining NCISM quota. The chip grid shows only the unallocated numbers in the reserved range, making it easy to pick the right beds.

Dept cards show 0 for NCISM Required / compliance rows missing ug_intake is 0 or NULL on the tenants row Set ug_intake in Supabase for the tenant. Compliance rows appear only when ug_intake > 0
Seeding NCISM Depts button silently does nothing All 10 mandatory depts already exist — Supabase unique constraint blocks duplicates Check the Departments tab. If some are missing, you can re-seed safely — only missing ones are inserted
Quick Setup Table 2 shows "GF" for all Building Sources rows Floor inputs in Table 1 were blank when beds were created — defaulted to 0 (GF) Enter floor values in Table 1 before clicking Auto-Distribute. Delete existing beds with 🗑 All Beds, then recreate via Create All
Block allocation is 38+22 instead of 30+30 Old type-first algorithm (before Session 74). Cached JS version Hard refresh. Current version uses block-first algorithm with _blkRemOffset rotating tie-breaker
Cannot insert bed — "violates check constraint beds_bed_type_check" Schema has old constraint without male_general / female_general Run the constraint update SQL from §9.4
Cannot delete a bed — "occupied beds cannot be deleted" Bed status = occupied; has a live IPD admission Change status to vacant in the edit drawer first (or discharge patient via ipd.html)
Building & Floor Breakdown cards show no data No beds have been created yet, OR beds were created without a ward_name Create beds via Quick Setup (which sets ward_name from block label) or edit existing beds to add a Ward Name

Page shows "Redirecting…" immediately Logged-in user's role is not super_admin or dept_admin Log in with a super_admin or dept_admin account. The page enforces role-based access at load time
Bed chips on dept card are not clickable Event propagation issue or old JS cache Hard refresh. Each chip has onclick="openBedDrawer('{id}')" — if it's silent, check the browser console for JS errors
